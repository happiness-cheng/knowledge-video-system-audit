# 视觉设计哲学核心

> 本文件是 Agent 每次正式进行视觉设计或实现前必须快速阅读的文件。

## 核心句

画面不是内容包装，而是与口播并行的视觉表达线。

解释决定画面有没有说东西；理解决定观众有没有听懂；吸引决定观众愿不愿意继续听。

声画可以重复信息，但不能重复劳动。

转场连接画面，Carry 连接意义。

视觉叙事让观众经历；视觉解释让观众看懂；编辑设计让观众看清结构；文字表演让观众感受到语气；真实证据让观众相信。

戏剧化负责让人感受到问题，隐喻负责让人理解问题，证据负责让人相信问题；三者都不能越过事实边界。

允许放大体验，不允许放大结论；允许借隐喻解释，不允许用隐喻代替证明；允许设计证据，不允许改变证据。

画面系统不能只继承设计结果，还要继承设计原理。

创意阶段对组件失明，实现阶段对能力清醒。

现有组件可以决定怎么做，不能决定做什么。

能力可以限制实现方式，不能擅自改写表达目标。

做不到可以谈，偷偷降级不可以。

视觉完成不以代码状态为准，而以观众实际接收到的表达为准。

## 有效画面的判断

以下画面形式可以成立：

- 静态画面；
- 纯文字画面；
- 编辑式、PPT 式画面；
- 动效少的画面。

判断依据不是外观，而是是否具有独立的解释、理解或吸引贡献。

删除后几乎没有损失的画面属于无贡献占位画面。

## 三类贡献

每个关键画面必须至少具备一项明确贡献：

- **解释贡献**：让观众看到口播无法单独传达的关系、过程、结构或证据；
- **理解贡献**：降低认知负荷，让复杂内容变得可理解；
- **吸引贡献**：产生继续观看的意愿，包括好奇、冲突、节奏变化或情绪推动。

三项都弱的画面应删除、合并或重写。

## 声画关系

声画各自承担什么必须明确。禁止理由只写"为了让观众看到口播内容"。

有效声画关系包括：

- 强化：画面让口播信息更具体；
- 互补：画面承担口播无法传达的部分；
- 对比：画面与口播形成张力；
- 证据：画面展示口播所引用的真实材料；
- 文字表演：文字按语气进行空间运动。

## 连续性

画面清空必须有叙事、认知或节奏理由，不能因为组件无法继承状态而默认重置。

关键 Shot 必须说明：

- 从上一个 Shot 带入什么；
- 本 Shot 改变什么；
- 传递给下一个 Shot 什么；
- 哪些对象持续存在；
- 重置的理由是什么。
