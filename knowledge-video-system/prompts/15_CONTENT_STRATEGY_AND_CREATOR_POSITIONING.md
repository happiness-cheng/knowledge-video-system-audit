# 内容账号策略与创作者定位

## 1. 账号定位

本账号不是单纯讲 Claude Code 的账号，而是记录如何通过第一手实验，把 AI 从陪聊工具变成学习、写作、开发和成长系统。

Claude Code 是重要工具，但不是唯一主角。

核心叙事方向：

- 我试了，你跟着试
- 我踩坑了，你避开
- 我发现机制了，你拿走用

不是"我教你用 AI"，而是"我试给你看"。

## 2. 内容比例

| 比例 | 类型                                  | 说明                           |
| ---- | ------------------------------------- | ------------------------------ |
| 70%  | 广痛点 + 可复制方案                   | 普通 AI 用户也能懂的问题和解法 |
| 20%  | Claude Code 事故复盘 + Agent 管理方法 | 真实翻车、调试过程、规则建立   |
| 10%  | 机制深度文                            | 背后原理、认知框架、系统设计   |

70/20/10 是月度/季度内容结构参考，不是单周硬性配额。当某个系列数据明显更强时，可以阶段性提高该方向占比。

## 3. 标题策略

### 优先使用

- 痛点 + 反差 + 解决结果
- "我以为是 A，后来发现是 B"
- 具体事故 + 更大结论

### 标题多用的词

听不懂 / 记不住 / 跑偏 / 乱改 / 偷懒 / 只会夸人 / 没有验证 / 不放过你 / 逼你讲出来

### 标题少用的词

Rules / Hook / Prompt / CLAUDE.md / Plan / Skill

原因：这些是工具内部术语，普通读者不关心。标题应该命中读者的感受，而不是工具的功能。

### 标题长度

8-14 个中文字主要适用于封面主标题、视频 S01 screenText 和短视频标题核心短句。

文章标题可以更长，但应通过冒号、破折号或副标题拆成"痛点短句 + 结果说明"，避免一整句平铺。例如：

```text
AI 一直夸你：不是你厉害，是它没验证
```

## 4. 视频前 5 秒规则

- 前 3 秒给强判断
- 前 5 秒给冲突
- 前 10 秒给例子或结果
- 不要用"最近我做了一个实验……"开头
- 优先用这种开头：

```text
如果 AI 一直夸你 Great，你要小心了。你可能根本没学会。
```

```text
你问 AI 越多，返工反而越多。
```

```text
我以为 prompt 写得好就够了，后来发现根本不是。
```

## 5. 发布前 6 问

每条内容发布前，必须回答：

1. 标题有没有读者能立刻感受到的痛点？
2. 痛点是不是普通 AI 用户也能懂？
3. 有没有真实经历？
4. 有没有反直觉结论？
5. 有没有具体方法？
6. 读者看完能不能拿走模板、流程、清单、规则片段或动作？

6 个问题中至少 4 个回答"是"，才值得发布。

## 6. 选题筛选

### 适合做的选题

- 我用 AI 做了一件事，结果出乎意料
- 我发现了一个常见错误，大多数人都在犯
- 我建立了一套方法，可以复制
- 我踩了一个坑，代价不小，但学到了

### 不适合做的选题

- 纯工具教程（"如何安装 XXX"）
- 纯功能介绍（"XXX 新功能来了"）
- 没有第一手经历的分析
- 没有可复制方法的观点
