# visualDirectionSpec.md

## 项目

视频标题：为什么每开一个新会话，都要重新介绍项目？

主题：xhs-white-editorial

目标：做成一条“视觉解释型知识视频”，不是文章压缩版，也不是自动播放 PPT。

核心视觉变化：

```text
用户在聊天窗口反复解释同一套项目规则
→ 发现这些规则只停留在当前聊天里
→ 散落在聊天记录里的项目知识
→ 汇聚成 CLAUDE.md 员工手册
→ 小尘从疑惑新人变成已入职同事
→ 项目地图点亮
→ 任务路径变短
```

## 总体导演原则

1. 画面要解释，不只是装饰。每个关键 scene 都必须有状态变化：反复解释→规则清空、散乱→有序、疑惑→明悟、绕路→直达、重复解释→默认共识、聊天记录→项目资产。
2. 小尘只作为图片素材，不作为视频人物层。禁止新增 characterLayer、personLayer、xiaochenAnimation 等未实现字段。
3. 手机端优先。主视觉中心明确，字幕不遮挡主体，方法页最终状态可截图保存。
4. S06/S07/S08/S10 是去 PPT 化重点。

## 小尘素材

- `xiaochen-confused-newcomer`：用于 S02，表现新会话像新人第一天上班，右上角问号。
- `xiaochen-reading-claude-handbook`：用于 S05/S08，表现小尘阅读 CLAUDE.md 员工手册，右上角灯泡。

## Scene 导演要求

### S01 Hook
主视觉：具体聊天窗口事件，不是抽象隐喻。用户提出需求后，Claude Code 反问项目背景、目录规则和验证方式；用户输入三条项目规则；`NEW CHAT` 后直接清空；用户换成“帮我加导出功能”，Claude 又重新追问项目结构、哪个目录能改、改完怎么测。不要抖动，不要先解释 CLAUDE.md，不要第一帧讲“办入职”。

### S02 痛点
承接 S01，解释“这些规则不是不存在，而是没有沉淀成每次新会话都能读取的项目规则”。可以用新员工类比辅助，但不要让“办入职”抢过前面的真实聊天症状。不要退化成普通左右清单。

### S03 知识放错地方
背景用弱聊天气泡/便签碎片；前景用项目背景、目录职责、历史坑三个标签。最后出现“知识放错地方”。不要展示大量聊天文字。

### S04 反转
先出现旧判断“它记性不好？”，再灰化或划掉，最后浮现“不是记忆问题 / 是入职问题”。下一页必须进入结构型类比。

### S05 新人先读员工手册
左侧现实新人流程，右侧 Claude Code 读 CLAUDE.md。使用 `xiaochen-reading-claude-handbook`。小灯泡可以出现但不能抢主标题。

### S06 沉淀成手册
全片核心转化页。左侧散落聊天碎片 / 便签，中间 CLAUDE.md 文件卡，右侧项目手册 / 默认共识。最低可接受实现：左侧节点逐步出现，中间 CLAUDE.md 放大并高亮，右侧结果最后出现，旧聊天碎片透明度下降。禁止只展示 CLAUDE.md 文字卡。

### S07 没有手册，路径绕远
要表现路径分叉、绕路、红色误判节点。没有真实截图也可以，但必须让观众看到“没有手册→路径绕远”。如果无法画路径，用错误节点递进替代。

### S08 读完手册，路径变短
右侧使用读手册图。表现小灯泡亮起、项目地图点亮、短路径直达目标。重点是“沟通成本下降”，不要说成代码质量突然飞跃。

### S09 边界
主结论：CLAUDE.md 是上下文，不是保险丝。画面用“手册提醒”和“权限锁”区分，出现 permissions.deny / PreToolUse hook，但不要塞太多官方细节。

### S10 模板
四层结构必须最终等权稳定可截图：身份偏好、行为准则、安全红线、纠错闭环。不要滚动长文本，手机端要能读清四项名称。

### S11 CTA
主行动：先写一份 CLAUDE.md。按钮文案：给 Claude Code 办入职。按钮需要轻微 pulse / 一闪一闪，强调“快去试试”；禁止高频闪烁，最后一秒稳定停住方便截图。

## Agent 执行提醒

- 本文件是导演说明，不是 videoSpec schema。
- 不得把 visualDirectionSpec 中的对象字段直接塞进 videoSpec。
- 小尘图片必须先进入 assetManifest，再被 videoSpec 的 assetLayout 引用。
- 如果小尘图片不存在，标记素材缺失，不得自行生成。
- 如果当前组件无法实现汇聚 / 路径 / 地图点亮，使用最低可接受实现，不得假装复杂动画已实现。
- 预览后必须输出 mobile_scaled_contact_sheet。
- command pass 不等于 visual pass。
