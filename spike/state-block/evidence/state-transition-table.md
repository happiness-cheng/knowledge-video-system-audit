# State Transition Table

## Pipeline 状态模型

```ts
type PipelineState = "idle" | "flowing" | "reviewing" | "blocked";
```

## 帧级状态转换

| 帧范围  | 时间  | PipelineState | 步骤1 内容候选 | 步骤2 审查 | 步骤3 音频 | 步骤4 字幕 | 步骤5 渲染 | 步骤6 发布 |
| ------- | ----- | ------------- | -------------- | ---------- | ---------- | ---------- | ---------- | ---------- |
| 0-29    | 0-1s  | idle          | 待机           | 待机       | 待机       | 待机       | 待机       | 待机       |
| 30-59   | 1-2s  | flowing       | 活跃           | 待机       | 待机       | 待机       | 待机       | 待机       |
| 60-119  | 2-4s  | reviewing     | 完成           | 活跃       | 待机       | 待机       | 待机       | 待机       |
| 120-300 | 4-10s | blocked       | 完成           | 失败(红叉) | 阻断(灰化) | 阻断(灰化) | 阻断(灰化) | 阻断(灰化) |

## 关键转换点

- **frame 120 (4s)**: reviewing → blocked
  - 触发条件: blockAtFrame = 120
  - 阻断效果: 步骤 3-6 同时进入阻断状态
  - 视觉表现: 红色背景、灰化滤镜、打叉图标、删除线、红光阴影
  - 连接箭头: 变为非活跃（暗淡）

## 阻断传播

阻断从 frame 120 开始，一次性影响所有 blockedAfter > 1 的步骤。不存在逐步传播——审查失败后所有下游同时冻结。

## 下游行为

- 下游步骤在阻断前：正常待机
- 下游步骤在阻断后：立即冻结，不可恢复
- 下游步骤不继续高亮、不继续推进、不显示完成状态
