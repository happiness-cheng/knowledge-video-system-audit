# Semantic Goal vs Actual

## 目标

审查失败后，下游生产真实停止。

## 实际画面

| 帧  | 时间 | 画面                      | 语义               |
| --- | ---- | ------------------------- | ------------------ |
| 30  | 1.0s | 步骤 1-2 正常推进         | 生产流开始         |
| 90  | 3.0s | 步骤 2（审查）高亮活跃    | 审查进行中         |
| 130 | 4.3s | 步骤 3-6 变红、灰化、打叉 | 阻断激活，下游冻结 |
| 200 | 6.7s | 下游保持冻结状态          | 状态持续继承       |
| 280 | 9.3s | 全流程可见，阻断仍生效    | 因果确认           |

## 冷观看验证

不看说明，只看短片：

1. 哪个节点失败了？→ 审查节点
2. 失败后发生了什么？→ 流程停止推进
3. 哪些步骤没有继续？→ 音频、字幕、渲染、发布
4. 停止是系统状态变化还是文字提醒？→ 系统状态变化（颜色、图标、灰化都变了）

## 结论

语义目标与实际画面一致。
