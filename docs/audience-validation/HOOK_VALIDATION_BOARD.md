# Hook 验证板：Claude Code 不是失忆，是没给它办入职

日期：2026-06-22  
主平台：抖音  
验证范围：信息 Hook、封面文案、前 15 秒内容承接、前 60 秒结构  
边界：本文件是创意验证板，不替换正式 `videoSpec.json`，不写入系统硬规则。

## 前置说明：本文件不是第一视觉验证

本文件解决的是“观众被画面停住之后，如何理解并继续看”。它属于第二层：

```text
视觉诱因
↓
信息 Hook
↓
证据与方法
```

第一层“第一帧是否本能吸引人”单独由 `VISUAL_TRIGGER_SHOT_LAB.md` 验证。  
如果视觉诱因不成立，不应靠增加标题、字幕或解释文字来补救。

## 验证目标

这轮不是证明“观众一定喜欢某种形式”，而是从现有高表现样本假设中挑出最值得承接强视觉开场的方向：

1. 普通痛点是否比 `CLAUDE.md` 文件名更适合作为第一信息。
2. 错误现场是否比孤立金句更适合作为视觉事件后的第一句解释。
3. “没手册 vs 有手册”的结果差异是否能让观众愿意继续看。
4. 可截图的“入职手册 4 层”是否能提供收藏动机。

## 观众假设

目标观众不是已经熟悉 Claude Code 配置的人，而是：

- 正在用 AI Coding 工具，但觉得它不稳定的人。
- 每次都要向 AI 重讲项目背景的人。
- 想要一个可照做方法，而不是听一段工具概念解释的人。

主假设：

> 抖音开头应先用强视觉事件让观众停住，再用“不是失忆，是没入职”命名这个事件。

当前最推荐的视觉事件是：

```text
项目世界正在运行
↓
NEW CHAT 被按下
↓
项目记忆全部归零
↓
DAY 1 出现
```

然后再进入本文件的信息 Hook。

## 封面方向

### 方向 A：反差判断型

```text
AI不是失忆
是没入职
```

副标题：

```text
Claude Code 总要你重复解释？
```

画面方向：

- 左侧：散落便签、重复解释、项目规则碎片。
- 右侧：一份清晰的“入职手册”。
- 情绪：原来问题不在 AI，而在我没把项目知识沉淀下来。

验证点：

- 观众是否能在 1 秒内理解“失忆”和“入职”的反差。
- 是否有继续看“怎么入职”的欲望。

风险：

- 如果只放这句，会像金句，痛点不够具体。
- 需要配合前 3 秒错误现场。

### 方向 B：行动阻断型

```text
别急着让AI干活
```

副标题：

```text
先给它一份入职手册
```

画面方向：

- 上半部分：用户把任务直接丢给 AI。
- 下半部分：AI 先读项目手册再执行。
- 情绪：别上来就开干，先补上下文。

验证点：

- 行动建议是否足够清楚。
- 是否比方向 A 更容易理解。

风险：

- 冲击力弱。
- 可能像普通效率建议，不如 A 有反差。

## 前 15 秒 Hook 版本

### Hook A：错误现场版（推荐）

核心逻辑：

```text
先让观众看见反复解释的痛点，
再浮现“不是失忆，是没入职”。
```

0-3 秒：

```text
项目做什么？
这个目录别动。
改完必须测试。
怎么又要重讲一遍？
```

3-7 秒：

```text
我一开始以为，是 Claude Code 失忆。
```

7-15 秒：

```text
后来发现不是。
是我根本没给它办入职。
```

画面：

- 重复解释的便签一条条压上来。
- 画面越来越乱。
- 最后这些便签被吸入一份“入职手册”。

验证点：

- 观众是否能先看到“我也这样”的痛点。
- “入职”隐喻是否因为前面的现场而变清楚。

### Hook B：结果差异版

核心逻辑：

```text
同一个任务，有无手册，结果完全不同。
```

0-3 秒：

```text
同一个修改任务，我试了两遍。
```

3-8 秒：

```text
没手册：它先绕一大圈。
有手册：它直接知道该看哪里。
```

8-15 秒：

```text
所以问题不是失忆。
是它还没入职。
```

画面：

- 左边路径绕远：猜目录、问规则、误改风险。
- 右边路径直达：读手册、定位目录、验证路线。

验证点：

- 观众是否因为“结果差异”愿意继续看证据。
- 是否比错误现场版更快建立可信度。

风险：

- 如果前 3 秒没有强痛点，可能像普通实验对比。

### Hook C：模板收益版

核心逻辑：

```text
给观众一个可拿走的结果。
```

0-3 秒：

```text
下次让 AI 改项目之前，
先给它这份入职手册。
```

3-8 秒：

```text
它至少要知道四件事：
身份、职责、路线、禁区。
```

8-15 秒：

```text
不然你每次都在重新培训一个新人。
```

画面：

- 一张四层模板卡快速展开。
- 每层只有一个大词。
- 最后回到“新人入职”的隐喻。

验证点：

- 收藏价值是否足够明确。
- 是否适合作为封面/开头，而不是中段方法页。

风险：

- 第一击情绪弱。
- 更适合 45-60 秒预告或中段方法页。

## 推荐前 60 秒结构

采用视觉诱因 + Hook A 为主，吸收 Hook B 的结果差异和 Hook C 的模板收益。

| 时间 | 观众任务 | 内容 | 画面 |
| --- | --- | --- | --- |
| 0-4s | 停下来 | 暂不依赖口播 | 项目世界运行，`NEW CHAT` 后全部归零，`DAY 1` 出现 |
| 4-8s | 命名痛点 | “我一开始以为，是 Claude Code 失忆。” | 归零后的项目世界旁出现“失忆？” |
| 8-15s | 反转判断 | “后来发现不是。是我没给它办入职。” | `DAY 1` 转化为“入职手册”入口 |
| 15-25s | 筛选观众 | “如果你也觉得它每次都像新人，这条是给你的。” | 反复解释、风格不一致、越改越乱三枚标签 |
| 25-45s | 建立信任 | “同一个任务，没手册先绕路，有手册直接定位。” | 左右路径对比：绕路 vs 直达 |
| 45-60s | 预告收藏 | “最后给你一份可抄的 Claude Code 入职手册结构。” | 四层模板预告：身份、职责、路线、禁区 |

## 推荐优先级

1. **先做 Visual Trigger Shot Lab 的 Shot A：项目世界重置**  
   它负责第一眼停留，不依赖观众先读懂文案。

2. **再用 Hook A 命名痛点**  
   它负责把视觉事件解释成“不是失忆，是没入职”。

3. **保留 Hook B 作为 25-45 秒证据段**  
   它负责让观众相信“这不是一句口号”。

4. **把 Hook C 放到 45-60 秒或中段模板页**  
   它负责收藏价值，不适合作为第一击。

## 通过标准

人工审片时只问三个问题：

1. 前 4 秒的视觉事件是否已经让人愿意停？
2. 15 秒时是否想知道“入职手册到底怎么写”？
3. 60 秒时是否已经相信这条视频会给一个可执行方法？

如果三个问题至少两个回答“是”，再进入 Remotion 前 60 秒画面原型。
