# Visual Trigger Shot Lab：强视觉诱因开场验证

日期：2026-06-22  
主平台：抖音  
验证范围：0-4 秒无声镜头  
边界：本文件只用于验证“第一眼是否愿意停”，不替换正式 `videoSpec.json`，不写入系统硬规则。

## 核心判断

当前系统已经能做“信息解释”，但抖音第一秒更需要先解决：

```text
观众还没理解内容时，画面本身值不值得停一下？
```

所以本轮先不验证完整逻辑，不验证口播，也不验证模板是否讲清楚。只验证一个问题：

> 第一帧和前 4 秒有没有足够强的视觉事件，让观众本能地想继续看？

## 设计原则

强视觉诱因不等于画面复杂。它至少要包含下面两类信号：

1. **高可见主体**：第一眼能看见一个明确主体，而不是一堆等权信息。
2. **未完成事件**：画面里有一件事正在发生，观众想看结果。
3. **状态风险**：危险、反常、坍塌、清空、误触、扩散、失控。
4. **强对比**：亮/灭、完整/崩解、正常/污染、有记忆/归零。
5. **镜头引导**：视线、路径、光、缩放或运动把注意力拉到同一点。

不要用“标题解释”承担第一击。文字可以出现，但只能在视觉事件抓住人之后补充含义。

## 通过标准

无字幕、无口播、静音播放时，人工只回答四个问题：

1. 第一帧是否有一个明确视觉主体？
2. 0-1 秒是否能看出“有事要发生”？
3. 1-4 秒是否出现了明显状态变化？
4. 看完后是否想知道“为什么会这样”？

至少 3 个回答“是”，这个 shot 才允许进入 Hook 文案和前 60 秒结构设计。

## Shot A：项目世界重置（推荐优先打样）

### 一句话

一个精密运行的项目世界，被 `NEW CHAT` 一按全部归零，中央重新出现 `DAY 1`。

### 第一帧

画面是一个俯视或斜俯视的“项目城市”：

- 目录树像建筑群一样发光。
- 文件依赖像道路和电路一样连接。
- 几条主路径正在稳定流动。
- 中央有一个小但醒目的按钮：`NEW CHAT`。
- 画面整体精密、有秩序、有规模感。

第一帧的视觉诱因：

- 大规模结构奇观。
- 发光路径形成视线引导。
- `NEW CHAT` 是一个明显会触发事件的按钮。

### 0-4 秒镜头

| 时间 | 事件 | 画面动作 |
| --- | --- | --- |
| 0-0.6s | 运行中 | 镜头轻微推进，路径流光，项目城市稳定发亮 |
| 0.6-1.2s | 按钮触发 | `NEW CHAT` 被按下，按钮周围出现短促白光 |
| 1.2-2.2s | 记忆断电 | 建筑灯光从远到近熄灭，连接线一批批消失 |
| 2.2-3.2s | 错误路线出现 | 一条红色路径从空白中误亮，绕向危险区域 |
| 3.2-4.0s | 身份归零 | 中央浮现 `DAY 1`，周围只剩几条残余红光 |

### Remotion 可执行方案

适合用 Remotion 直接完成：

- SVG/Canvas 绘制节点、路径、目录建筑。
- `interpolate()` 控制路径亮度、透明度和镜头推进。
- `spring()` 控制按钮按下、震动和 `DAY 1` 弹出。
- 分层实现：背景网格、建筑节点、路径流光、按钮、红色危险路径、中央文本。

适合用素材补足：

- 如果要更电影感，可先生成一张“项目城市”底图，再用 Remotion 叠加可控路径、灯光和按钮。

### 风险

- 如果节点太多，可能变成装饰噪声。
- 如果 `NEW CHAT` 太小，观众不知道触发点。
- 如果重置前后对比不够强，会变成普通科技背景。

### 验证问题

- 第一帧有没有“精密系统正在运行”的感觉？
- 按下 `NEW CHAT` 后，是否能感到“整个项目记忆被清空”？
- `DAY 1` 是否自然引出“每次新会话都是新人”？

## Shot B：错误指令扩散

### 一句话

一条红色错误指令像污染一样沿项目结构扩散，正常节点被逐个染红。

### 第一帧

- 画面中心是一行短指令或一个红色命令块。
- 周围是清晰的项目结构网络。
- 红色命令块旁边有一个小小的执行光标或运行按钮。

第一帧的视觉诱因：

- 红色危险源。
- 即将执行的动作。
- 周围结构暗示后果很大。

### 0-4 秒镜头

| 时间 | 事件 | 画面动作 |
| --- | --- | --- |
| 0-0.5s | 危险源出现 | 红色命令块轻微闪烁，光标停在执行点 |
| 0.5-1.2s | 开始执行 | 一道红光进入最近的文件节点 |
| 1.2-2.5s | 扩散 | 红色沿连接线蔓延，正常节点逐个变色 |
| 2.5-3.4s | 失控 | 多条路径同时爆亮，画面短暂震动 |
| 3.4-4.0s | 留悬念 | 画面定格在“危险扩散一半”的状态 |

### Remotion 可执行方案

- 用 SVG path dashoffset 做红光沿路径移动。
- 用节点数组控制污染状态。
- 用局部震动和红色 glow 表达失控。
- 不需要复杂素材，适合纯 Remotion 打样。

### 风险

- 容易变成网络安全/病毒题材，偏离 Claude Code 入职手册。
- 如果没有后续解释，观众可能不知道这是“AI 不懂项目边界”。

### 验证问题

- 红色扩散是否足够本能吸睛？
- 观众是否能感到“这个操作会带来后果”？
- 是否会误解成安全攻击视频？

## Shot C：新人误触核心

### 一句话

一个“第一天上班”的 AI 标记正在接近发红光的核心文件，危险动作尚未完成。

### 第一帧

不强制使用人物。可选主体：

- 人物手部近景。
- 机械手。
- AI 光标。
- 带 `DAY 1` 胸牌的抽象角色标记。

画面中心是发红光的核心对象，例如：

```text
core/config
rules
production
```

第一帧的视觉诱因：

- 明确主体正在靠近危险目标。
- 红光目标天然吸引注意。
- 动作未完成，观众想看会不会碰到。

### 0-4 秒镜头

| 时间 | 事件 | 画面动作 |
| --- | --- | --- |
| 0-0.8s | 接近核心 | 主体从画面边缘进入，朝红光核心移动 |
| 0.8-1.6s | 危险提示 | 核心周围出现细小 warning ring |
| 1.6-2.4s | 即将误触 | 主体停在核心边缘，红光增强 |
| 2.4-3.2s | 身份暴露 | 镜头拉近 `DAY 1` 标记 |
| 3.2-4.0s | 悬念定格 | 主体距离核心只剩一点，画面暂停 |

### Remotion 可执行方案

- 如果用抽象光标/机械手，Remotion 可直接做。
- 如果用高质量人物，需要生成图或角色关键帧负责主视觉，Remotion 负责镜头推进、红光、警告环和局部运动。

### 风险

- 人物路线质量门槛高，低质量会显得廉价。
- 人物太抢眼但与知识主题弱相关，会吸引错误受众。
- 抽象主体太弱，则不如 Shot A 有奇观感。

### 验证问题

- 危险动作是否一眼可读？
- `DAY 1` 是否和“新人误触核心”自然绑定？
- 是否值得作为账号代表作级高级开场，而不是普通打样？

## 推荐打样顺序

1. **Shot A：项目世界重置**  
   当前最适合先做。它有奇观感，又更符合 Remotion 的结构化动画优势。

2. **Shot B：错误指令扩散**  
   适合作为低成本纯 Remotion 对照，验证“危险扩散”是否比“项目重置”更抓人。

3. **Shot C：新人误触核心**  
   创意吸引力高，但依赖主视觉质量。建议等 Shot A/B 验证后再决定是否投入。

## 与信息 Hook 的关系

视觉诱因通过后，才进入信息 Hook：

```text
视觉事件：NEW CHAT 让项目世界归零
↓
信息命名：不是 AI 失忆，是每次都重新入职
↓
方法承诺：给它一份 Claude Code 入职手册
↓
证据结构：没手册绕路，有手册直达
```

如果视觉诱因不成立，不要靠加更多文字救开场。应该先换第一视觉主体或视觉事件。
