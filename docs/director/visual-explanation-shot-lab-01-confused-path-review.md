# Visual Explanation Shot Lab 01 — Confused Path 审查报告

**日期**: 2026-06-18
**原型**: PromptCompletionExplainer v3 (Confused Path)
**位置**: `src/video-system/experiments/shot-lab/PromptCompletionExplainer.tsx`
**关键帧**: `out/shot-lab-01-frames/*-v3.png` 和 `*-final.png`

## 1. 右侧初始状态是否从"看不见"变成"可见但混乱"？

**是，明显改进。**

原版（Hardened）：
- 右侧是灰色模糊 radial-gradient + blur
- 中央半透明文字"回答模糊 · 无法使用"
- 几乎看不见，观众不知道右侧在表达什么

新版（Confused Path）：
- 右侧是一条弯曲的 SVG 路径，从左下到右上
- 路径上有 2 条红色岔路（表示走偏）
- 路径周围有 4 个碎片文字（"大概..."、"可能..."、"一般来说"、"某种程度"）
- 终点是一个模糊的"泛泛回答"块
- **观众能明确看到：AI 有输出，但方向混乱、不能用**

关键帧对比：
- `01-confused-path-v3.png` (frame 45): 路径开始绘制
- `02-confused-path-drawn.png` (frame 120): 乱麻路径完全显示

## 2. 是否表达出"一头乱麻 / 走错路"的感觉？

**是，通过三个视觉元素表达：**

1. **弯曲路径**: 路径有 4 个控制点，形成明显的弯曲，表示 AI 在绕路
2. **红色岔路**: 2 条红色虚线岔路，表示 AI 走偏了方向
3. **碎片文字**: "大概..."、"可能..."、"一般来说"、"某种程度"，表示 AI 在输出模糊、不确定的内容

**效果**: 观众看到的是一个"混乱的思考过程"，而不是"没有输出"。

## 3. 条件补充是否真的让路径逐步变正确？

**是，通过路径校正动画实现。**

Phase 2（上下文飞入）的路径校正：
- 每个上下文模块飞入时，路径控制点被插值校正
- 路径从弯曲逐渐变直
- 岔路 opacity 逐渐降低（消失）
- 路径颜色从灰色变为紫色（表示变清晰）

关键帧对比：
- `03-guided-path-correction.png` (frame 180): 路径正在校正，仍有弯曲
- `04-guided-path-halfway.png` (frame 250): 路径半直，岔路几乎消失

**校正逻辑**：
```
correctionProgress = phase2Frame / 180  // Phase 2 内逐渐校正
// 每个上下文模块贡献 25% 的校正进度
```

## 4. 最终结构是否从路径中长出来，而不是突然列表出现？

**是，结构从路径终点生长出来。**

Phase 3-4 的结构生长：
- 路径完全变直后，结构化回答从路径终点开始生长
- 四段结构化回答逐条滑入，起点是路径终点
- 结构化回答的容器从模糊变清晰（blur 降低）

关键帧对比：
- `05-structure-forming.png` (frame 400): 结构正在形成，仍有模糊
- `06-structure-clear.png` (frame 500): 结构完全清晰

**效果**: 观众看到的是"结构从路径中自然生长"，而不是"突然出现一个列表"。

## 5. 是否仍保持用户喜欢的整体感觉？

**保持，且增强了。**

用户喜欢的核心感觉：
- ✅ 画面本身在讲解，不是排版文字
- ✅ 左侧 prompt 被补齐，右侧结果随之变清晰
- ✅ 中间因果箭头
- ✅ 结论："AI 没变，材料变完整了"

新增的增强：
- ✅ 右侧从"看不见"变成"可见但混乱"，观众能理解 AI 在走弯路
- ✅ 路径校正动画让"补齐 → 变清晰"的因果关系更直观
- ✅ 结构从路径中生长，比突然出现列表更有说服力

## 6. 是否比 hardening 版本更接近"画面在讲解"？

**是，明显更接近。**

Hardening 版本的问题：
- 右侧灰雾几乎看不见，观众不知道在表达什么
- 结构化回答突然出现，没有解释"为什么是这四段"

Confused Path 版本的改进：
- 右侧乱麻路径让观众看到"AI 在走弯路"
- 路径校正动画让观众看到"上下文在纠正 AI 方向"
- 结构从路径中生长，让观众看到"清晰输出是从正确路径中长出来的"

**结论**: 画面真正参与了讲解，而不是把文字排版在屏幕上。
