# Visual Explanation Craft Rules v0

**日期**: 2026-06-18
**来源**: PromptCompletionExplainer 原型验证 + 用户 Studio 确认
**适用范围**: 所有 Visual Explanation Shot Pattern 的 craft pass

---

## 1. 核心原则

### 1.1 画面本身参与讲解

画面不是旁白的配图，画面本身在传递因果关系。观众即使静音观看，也能从画面变化中理解核心逻辑。

**检验标准**: 关掉声音，观众能否理解"发生了什么变化"和"为什么会这样"？

### 1.2 文字是标签，不是主体

文字用于标注画面中的对象，帮助观众快速识别。文字不是信息的主体，画面变化才是。

**违反示例**: 全屏大字列出 4 个要点 → 这是 PPT，不是视觉解释。
**正确示例**: 4 个对象从模糊变清晰，每个对象旁边有小标签 → 画面在讲解。

### 1.3 变化过程比静态结果更重要

观众需要看到"从 A 到 B"的变化过程，而不是直接看到结果 B。变化过程本身就是解释。

**检验标准**: 如果去掉变化动画，只保留最终状态，观众是否还能理解因果关系？如果能，说明动画是装饰；如果不能，说明动画在讲解。

### 1.4 每个动作必须对应一个理解动作

每个视觉变化（飞入、消失、变直、展开）必须对应一个理解动作（补齐、减少、校正、揭示）。不服务理解的动效应降级或移除。

**检验标准**: 这个动画让观众理解了什么？如果答不上来，这个动画就是装饰。

## 2. 视觉中心规则

### 2.1 每一帧必须有唯一视觉中心

观众的眼睛应该看向哪里，必须由画面设计决定。不能让观众自己决定看哪里。

**实现方式**:
- 通过大小对比（大对象吸引注意力）
- 通过颜色对比（高饱和度吸引注意力）
- 通过运动对比（正在动的对象吸引注意力）
- 通过位置（中心偏上区域是默认视觉中心）

### 2.2 视觉中心必须对应当前口播或理解任务

如果旁白在说"补充上下文"，视觉中心应该在上下文模块飞入的位置，而不是在其他地方。

**违反示例**: 旁白说"身份信息补充进来"，但观众在看右侧的路径变化 → 视觉中心错位。
**正确示例**: 旁白说"身份信息补充进来"，视觉中心跟随上下文模块飞入 → 视听同步。

### 2.3 不能让辅助卡片长期抢主视觉

辅助信息（标签、说明、背景装饰）可以出现，但不能长期占据视觉中心。如果辅助信息比主解释对象更吸引眼球，需要降级。

**降级方式**:
- 降低 opacity（0.3-0.5）
- 缩小尺寸
- 移到边缘位置
- 减少动画幅度

## 3. 因果动线规则

### 3.1 必须明确输入、变化、输出

视觉解释镜头必须有三个区域：
- **输入区**: 原始状态（模糊问题、不完整信息）
- **变化区**: 转换过程（路径校正、模块飞入、模糊消失）
- **输出区**: 最终状态（清晰结构、完整回答）

### 3.2 输入和输出之间必须有因果连接

不能只是左右并列两个区域，必须有中间的因果连接。连接方式可以是：
- 路径/线条
- 箭头/指向
- 动态过渡（输入变化驱动输出变化）
- 空间流动（从左到右、从上到下）

**违反示例**: 左侧模糊问题 + 右侧灰雾 → 只是并列，没有因果连接。
**正确示例**: 左侧模糊问题 → 中间路径 → 右侧灰雾 → 路径在传递因果关系。

### 3.3 中间区域应承担"变化 / 校正 / 转换"的功能

中间区域不是空白，而是因果关系的载体。它应该展示：
- 材料在补齐（模块飞入）
- 方向在校正（路径变直）
- 不确定性在减少（模糊词消失）
- 结构在形成（块状展开）

## 4. 字体层级规则

### 4.1 三级字体层级

| 层级 | 用途 | 建议字号（1080p） | 建议字号（mobile-scaled） |
|------|------|-------------------|--------------------------|
| L1 核心结论 | 最终结论、核心观点 | 40-52px | 24-32px |
| L2 关键对象 | 对象标签、模块名称 | 22-28px | 14-18px |
| L3 辅助说明 | 说明文字、状态提示 | 14-18px | 10-12px |

### 4.2 关键语义词不能小到手机端不可读

观众在手机竖屏观看时，画面被缩放到约 360px 宽。关键语义词（结论、对象名称）必须在这个尺寸下仍然清晰可读。

**mobile-scaled 检查**: 渲染 1080p 画面 → 缩放到 360px 宽 → 检查关键文字是否可读。

### 4.3 所有关键文字必须通过 mobile-scaled 检查

**检查清单**:
- [ ] L1 核心结论在 360px 宽下是否清晰可读？
- [ ] L2 关键对象标签在 360px 宽下是否可读？
- [ ] L3 辅助说明在 360px 宽下是否可辨认（不要求清晰）？
- [ ] 是否有文字被裁切或溢出？

## 5. 布局规则

### 5.1 重要对象优先放在手机端中心安全区

手机竖屏观看时，画面中心区域（约 60% 宽度 × 60% 高度）是最安全的可视区域。重要对象应优先放在这里。

**安全区**: 1080p 画面中，约 (216, 216) 到 (864, 864) 的区域。

### 5.2 16:9 横版也要考虑手机竖屏观看时的投影大小

虽然视频是 16:9 横版，但手机用户可能竖屏观看。此时画面被缩放到约 360px × 202px。重要对象不能太小，否则在投影中不可见。

**最小对象尺寸**: 关键对象在投影中至少 30px × 30px。

### 5.3 左右结构中，主解释对象不能被放得过小或过远

左右结构是常见的布局方式，但左右两个区域不能离得太远，否则观众的视线需要大幅移动。

**建议**: 左右区域间距不超过画面宽度的 10%（约 100px @ 1080p）。

### 5.4 留白必须服务动线，不是单纯空

留白的目的是：
- 引导视线流动
- 给动效留出空间
- 避免信息过载

如果留白只是"空"，没有服务动线，就是浪费空间。

## 6. 运动语义规则

### 6.1 运动必须对应理解动作

| 运动类型 | 对应理解动作 | 示例 |
|----------|--------------|------|
| 模块飞入 | 材料补齐 | 上下文标签从左下飞入 |
| 路径校正 | 理解方向变正确 | 弯曲路径逐渐变直 |
| 模糊词消失 | 不确定性减少 | "大概..."逐渐淡化 |
| 结构展开 | 输出变清晰 | 模糊块变成清晰列表 |
| 发光/高亮 | 到达/确认 | 模块到达目标后发光 |
| 路径绘制 | 过程展开 | 乱麻路径逐渐绘制 |

### 6.2 不服务理解的装饰动效应降级

以下动画如果不对应理解动作，应降级或移除：
- 闪光效果（除非对应"确认/到达"）
- 背景粒子（除非对应"散开/消失"）
- 渐变色变化（除非对应"状态转换"）
- 装饰性旋转（除非对应"方向变化"）

### 6.3 动画节奏必须与口播同步

动画的触发时机应与旁白的节奏匹配：
- 旁白说"身份信息补充进来" → 身份模块开始飞入
- 旁白说"方向变明确了" → 路径开始校正
- 旁白说"结果变清晰了" → 结构开始展开

**检验标准**: 静音观看时，动画节奏是否自然？如果动画太快或太慢，说明与口播不同步。

## 7. 去 PPT 化检查

### 7.1 检查清单

每个 Visual Explanation Shot 必须通过以下检查：

**卡片检查**:
- [ ] 是否只是卡片出现？→ 必须有变化过程
- [ ] 是否只是列表展示？→ 必须有动态揭示

**文字检查**:
- [ ] 是否突然切到大字结论页？→ 结论必须从画面中浮现
- [ ] 是否观众不看字幕就无法理解主要变化？→ 画面本身必须能讲解

**运动检查**:
- [ ] 是否画面不动但旁白在讲？→ 画面必须与旁白同步变化
- [ ] 是否动画只是装饰？→ 每个动画必须对应理解动作

**布局检查**:
- [ ] 是否左右并列没有因果连接？→ 必须有中间因果动线
- [ ] 是否重要对象被放得过小？→ 必须通过 mobile-scaled 检查

### 7.2 常见 PPT 化错误

| 错误类型 | 表现 | 修复方式 |
|----------|------|----------|
| 静态卡片 | 信息卡片直接出现，没有变化 | 添加揭示动画，让卡片从模糊变清晰 |
| 列表展示 | 4 个要点逐条滑入 | 改为动态展开，或从路径中生长 |
| 大字结论 | 全屏大字 + 副标题 | 结论从画面内自然浮现 |
| 画面静止 | 旁白在讲，画面不动 | 添加与旁白同步的视觉变化 |
| 左右并列 | 输入和输出只是左右排列 | 添加中间因果连接（路径、箭头、动态过渡） |
