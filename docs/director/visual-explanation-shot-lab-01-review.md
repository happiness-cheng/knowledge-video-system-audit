# Visual Explanation Shot Lab 01 — 原型审查报告

**日期**: 2026-06-18
**原型**: PromptCompletionExplainer (630 frames / 21s / 30fps)
**位置**: `src/video-system/experiments/shot-lab/PromptCompletionExplainer.tsx`

## 1. 为什么用户认为这个方向成立？

**核心判断: "画面本身在讲解，而不是把文字排版在屏幕上。"**

当前系统的所有场景（bullets、comparison、timeline 等）本质是信息卡片——观众读完文字就结束了。这个原型的区别在于：

- **观众看到的是一个变化过程**，而非静态信息
- **动画传递因果关系**：材料补齐 → 结果清晰，这个"动"本身就是解释
- **灰雾散开是最成功的视觉隐喻**：模糊 = 信息不足，这个直觉不需要文字解释

用户确认"这个画面就是我想要的感觉"，说明视觉解释的方向值得继续。

## 2. 哪些画面元素真正帮助理解？

| 元素 | 帮助程度 | 原因 |
|------|----------|------|
| 灰雾模糊 → 渐渐清晰 | **高** | 直观展示"信息不足 = 结果模糊"的因果关系，观众不需要读文字就能理解 |
| 上下文模块飞入问题框 | **高** | 观众看到"材料在填充"，理解补齐的过程是动态的 |
| 虚线变实线 | **中** | 边框变化暗示"不完整 → 完整"，但不如灰雾直观 |
| 拼图闪光 | **低** | 偏装饰，闪光本身不传递信息，可以去掉 |
| 四段结果逐条滑入 | **低** | 仍然是列表展示，没有解释"为什么是这四段" |

**结论: 灰雾 + 飞入是最核心的两个视觉解释手法，其他元素是辅助或装饰。**

## 3. 哪些地方仍可能像 PPT？

| 位置 | PPT 感来源 | 改进方向 |
|------|-----------|---------|
| Phase 5 结论页 | 全屏大字 + 副标题 = ending slide | 考虑用动画收尾，而非静态文字 |
| 四段结果列表 | 图标 + 标题 + 序号 = 信息卡片 | 探索更动态的揭示方式（如拼图块亮起） |
| Phase 标签 | 左上角"PHASE 01" = 演示文稿提示 | 正式视频应移除，用画面过渡自然切换 |

**判断: Phase 5 是最像 PPT 的部分，需要在进入实践版前重新设计。**

## 4. 这个 pattern 能否复用于 AI 提问类视频？

**可以复用，但需要泛化。**

当前原型硬编码了"帮我写一下"的具体场景，但核心 pattern 是通用的：

```
模糊/不完整的输入 → 补充上下文 → 输出变清晰
```

这个 pattern 适用于：
- Prompt 工程教学（"为什么你的 prompt 不好用"）
- AI 提问优化（"问 AI 的正确姿势"）
- 信息完整性对结果的影响（"输入决定输出"）

**复用条件:**
- 内容需要参数化（所有文字改为 props）
- 视觉风格需要适配不同主题
- 结果展示需要更动态

## 5. Level 评估：Level 2 组合能力 vs Level 3 migration candidate？

**结论: Level 2 组合能力候选。**

| 评估维度 | Level 2 | Level 3 |
|----------|---------|---------|
| 复用性 | 可组合到不同视频 | 需要独立迁移 |
| 参数化程度 | 需要先抽象 | 已经参数化 |
| 视觉风格 | 需要适配 | 已经多主题 |
| 当前状态 | ❌ 硬编码 | ❌ 单一风格 |

**理由:**
- 核心 pattern（左侧因 → 右侧果同步变化）可以抽象为 `CauseEffectPair` 组件
- 但当前实现过于具体，需要先完成参数化和主题适配
- 建议先作为 Level 2 组合能力使用，验证效果后再考虑 Level 3 迁移

## 6. 是否建议用它重做"为什么你用 AI 总是不满意？"中的核心段落？

**建议: 可以尝试，但需要先做参数化。**

"为什么你用 AI 总是不满意？"的核心论点是：
- 用户提问太模糊（"帮我写一下"）
- AI 无法理解真正需求
- 补充上下文后输出变好

这与 PromptCompletionExplainer 的 pattern 完全吻合。但当前原型硬编码了具体场景，需要先完成：

1. **参数化**: 所有文字内容改为 props
2. **主题适配**: 至少支持白底杂志风（xhs-white-editorial）
3. **移除调试元素**: Phase 标签、闪光效果
4. **重新设计结论页**: 用动画替代全屏大字

**如果完成以上工作，这个 pattern 是重做该视频核心段落的最佳候选。**
